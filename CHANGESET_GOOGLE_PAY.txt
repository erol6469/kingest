Google Pay a été intégré dans l'app Kingest comme méthode de paiement sécurisée 
utilisant Stripe comme gateway de tokenisation. L'intégration supporte:
- Paiement Google Pay réel sur Android avec isReadyToPay()
- Fallback simulation mode sur iOS WKWebView
- Badge "Disponible" quand Google Pay est détecté
- Rate limiting et validation des montants
- Métadonnées de transaction pour audit

DÉTAILS TECHNIQUES
================================================================================

FRONTEND CHANGES:
─────────────────

1. Google Pay API Loading:
   - Script loaded asynchronously in <head>
   - Doesn't block page load
   - Available after ~500ms (checked in useEffect)

2. Google Pay Detection:
   - Checks window.google.payments.api.PaymentsClient existence
   - Calls isReadyToPay() with TEST environment
   - Supports CARD payment method with VISA/MASTERCARD
   - PAN_ONLY and CRYPTOGRAM_3DS auth methods
   - Sets googlePayAvailable state if ready

3. Payment Request:
   - PaymentDataRequest with TEST environment
   - Stripe gateway with version 2024-04-10
   - Publishable key fetched from /api/stripe/config
   - Transaction info: amount in EUR
   - Merchant info: kingest_app

4. Payment Flow:
   a) User selects Google Pay method
   b) User enters amount >= 10 EUR
   c) Clicks "Confirmer"
   d) loadPaymentData() shows Google Pay popup
   e) User completes payment
   f) Token sent to backend
   g) Backend processes via Stripe

5. Badge Display:
   - Shows "Disponible" in green when googlePayAvailable === true
   - Appears under Google Pay in methods list
   - Same styling as Apple Pay badge

6. Fallback Behavior:
   - If window.google.payments.api.PaymentsClient not available
   - Simulation mode: 2 second delay
   - Amount credited automatically
   - No banner message (silent fallback)

BACKEND CHANGES:
────────────────

1. New Endpoint: POST /api/stripe/google-pay
   - Rate Limiter: 10 requests per 15 minutes
   - Accepts: token, amount, currency
   - Returns: { ok: true, id, transactionId } or { ok: false, error }

2. Token Processing:
   - Receives: JSON stringified Stripe payment method
   - Parses to extract: paymentMethodData.tokenizationData.token
   - Extracts Stripe payment method ID from parsed JSON
   - Validates: token exists and is parseable

3. PaymentIntent Creation:
   - Amount: converted to cents (Math.round)
   - Currency: EUR (or from request)
   - Payment method: Stripe token ID from Google Pay
   - Confirm: true (immediate confirmation)
   - Metadata:
     * source: "google_pay"
     * platform: "kingest_ios"

4. Success Response:
   - Checks: paymentIntent.status === "succeeded"
   - Records transaction:
     * id: UUID
     * type: "deposit"
     * method: "google_pay"
     * status: "completed"
     * stripeId: paymentIntent.id
   - Returns: { ok: true, id, transactionId }

5. Error Handling:
   - Invalid parameters: "Invalid parameters"
   - Token parsing failed: "Token parsing failed"
   - Invalid token format: "Invalid token format"
   - Stripe errors: Stripe error message
   - Pending payment: "Payment requires additional action"

================================================================================
SÉCURITÉ
================================================================================

✅ Stripe Tokenization
   - Clé publique Stripe utilisée pour tokeniser les cartes
   - Pas de numéro de carte transmis au backend
   - Token Stripe opaque et jetable

✅ Rate Limiting
   - 10 requêtes par 15 minutes par IP
   - Protège contre les attaques par force brute

✅ Amount Validation
   - MIN_AMOUNT: 10 EUR
   - MAX_AMOUNT: 50000 EUR
   - Protège contre les montants invalides

✅ Metadata Recording
   - Source enregistrée: "google_pay"
   - Platform enregistrée: "kingest_ios"
   - Permet l'audit et le suivi

✅ No Sensitive Data
   - Aucune donnée PII stockée
   - Aucun numéro de carte
   - Aucun token sensible

✅ HTTPS Only
   - Google Pay refuse HTTP
   - Transmission chiffrée
   - Certificat SSL/TLS requis
